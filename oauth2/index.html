OAuth2

OAuth is an open standard for authorization. OAuth provides a method for clients to access server resources on behalf of a resource owner.

OAuth gives the ability for client applications to access resources on a server owned by a user.

OAuth2

Solves the authentication problem that plagued the internet.

Now, instead of giving your regular keys to the valet, you give them a valet key, which only permits them to do certain things that you authorize (by giving them the key).

OAuth2 allows you to login to one service, using your identity with another service, securely.

OAuth2 defines four roles:

OAuth2 Example: GitHub

  1. User clicks button to login with GitHub
  2. User is redirected to https://github.com/login/oauth/authorize
  3. User logs in (if not already), then confirms they want to grant you access to their account
  4. User is redirected back to your service
  5. You then make a POST request with the supplied authorization grant to get an access token:
    POST https://github.com/login/oauth/access_token
  6. The response, if the request was valid, contains an access_token
